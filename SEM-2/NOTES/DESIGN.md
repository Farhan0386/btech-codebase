# UI/UX DESIGN, DESIGN THINKING & FIGMA — EXAM ANSWERS

## SECTION A — SHORT ANSWERS

1. Explain the difference between UI and UX using a real-world example.

Answer:
- **UI (User Interface)** is the visual and interactive surface of a product — buttons, colors, layout.
- **UX (User Experience)** is the overall feeling and ease of use when a user accomplishes tasks.
- Example: In a **mobile banking app**, UI = the visual design of transfer screen (fields, icons); UX = how quickly and confidently a user completes a fund transfer (flow, feedback, errors).
- **Key point:** Good UI helps UX, but excellent UX may require interaction design, content, and service design beyond visuals.

2. What are the five phases of Design Thinking? Briefly describe what happens in each phase.

Answer:
- **Empathize:** Understand users via interviews, observation, and research to gather needs and pain points.
- **Define:** Synthesize research into clear **problem statements** and personas (focus the challenge).
- **Ideate:** Generate many creative solutions using brainstorming, SCAMPER, card sorting.
- **Prototype:** Build cheap, fast representations (paper/wireframes) to explore ideas.
- **Test:** Validate prototypes with users, collect feedback, iterate back to earlier phases.

3. Write a proper problem statement for this scenario:
   “College students struggle to find internship opportunities relevant to their field.”

Answer:
- **Problem Statement (Good):** "College students in STEM fields lack a centralized, filtered platform to discover relevant internships aligned to their major and skills, causing missed opportunities and low placement rates." 
- **Includes:** target users (**college students**), problem (**finding relevant internships**), impact (**missed opportunities, low placements**).

4. What is the difference between a User Journey Map and a User Flow?

Answer:
- **User Journey Map:** Narrative of user’s emotional and contextual experience across touchpoints over time (includes feelings, pain points).
- **User Flow:** Step-by-step diagram showing screens, decisions, and actions to complete a task (focuses on navigation and states).
- **Example:** Journey map shows student’s emotions when applying for internships; user flow shows screens from search → filter → apply.

5. List and explain three ideation techniques.

Answer:
- **Brainstorming:** Rapid idea generation in a group; encourage quantity, defer judgment; use prompts to diversify ideas.
- **SCAMPER:** Modify existing ideas by Substitute, Combine, Adapt, Modify, Put to another use, Eliminate, Reverse.
- **Crazy 8s:** Fast individual sketching technique (8 ideas in 8 minutes) to force divergent thinking and visual concepts.

6. What is a Persona? What makes a good persona different from a bad one?

Answer:
- **Persona:** A fictional, research-based representation of a target user including goals, behaviors, needs, and context.
- **Good persona:** Based on real research, includes demographics, goals, frustrations, scenarios, and a photo; actionable for design decisions.
- **Bad persona:** Generic stereotypes, no data, vague needs, not tied to measurable behaviors — not useful for design.

7. Explain the concept of mental models with an example.

Answer:
- **Mental model:** The internal representation users form about how a system works, guiding their expectations and actions.
- **Example:** Users expect a shopping cart icon to show items; if checkout flow violates this model, users get confused.
- Designers map mental models to interface models to minimize mismatch and improve usability.

8. What is an Empathy Map? Describe all six sections.

Answer:
- **Empathy Map:** A collaborative tool to capture user insights and viewpoint.
- **Six sections:**
  - **Says:** Direct quotes from users.
  - **Thinks:** User thoughts and beliefs (often unspoken).
  - **Does:** Observable user actions.
  - **Feels:** Emotions and mood during experience.
  - **Pains:** Frustrations and obstacles.
  - **Gains:** User goals and benefits sought.

9. What are Nielsen’s 10 Usability Heuristics? Explain any five with examples.

Answer:
- **List (names):** Visibility of system status; Match between system and real world; User control and freedom; Consistency and standards; Error prevention; Recognition rather than recall; Flexibility and efficiency of use; Aesthetic and minimalist design; Help users recognize, diagnose, and recover from errors; Help and documentation.
- **Five explained:**
  - **Visibility of system status:** Show progress (e.g., upload progress bar) so users know system state.
  - **Match between system and real world:** Use user language and metaphors (folder icon for files).
  - **Error prevention:** Disable invalid actions instead of showing errors (grey out submit until fields complete).
  - **Recognition rather than recall:** Provide visible options and history (autocomplete search).
  - **Aesthetic and minimalist design:** Avoid clutter; show only relevant controls on each screen.

10. What is the difference between a Wireframe, Mockup, and Prototype?

Answer:
- **Wireframe:** Low-fidelity blueprints showing layout and hierarchy without final visuals; focus on content and structure.
- **Mockup:** High-fidelity static visual design; shows colors, typography, and UI elements but not interactive.
- **Prototype:** Interactive model (low to high fidelity) that simulates user flow and interactions for testing.

11. What is a Frame in Figma and how is it different from a Group?

Answer:
- **Frame:** A container used for artboards, responsive layouts, and constraints — can have background, overflow behavior, and layout features.
- **Group:** Simple grouping of layers for organization; no layout controls or responsive constraints.
- **Key:** Frames are structural (used for screens), groups are organizational.

12. What is Auto Layout in Figma? Name any two properties you can control.

Answer:
- **Auto Layout:** A layout system that auto-sizes and positions children in directions (vertical/horizontal), useful for responsive components.
- **Two properties:** Padding (inside spacing), spacing between items; also alignment and resizing behavior.

13. What are Constraints in Figma and why are they important?

Answer:
- **Constraints:** Rules that determine how child layers behave when parent frames resize (e.g., left, right, center, scale).
- **Importance:** Enable responsive designs and predictable behavior for different screen sizes and during handoff to developers.

14. What is the difference between a Local Style and a Shared Library in Figma?

Answer:
- **Local Style:** Styles (colors, text, effects) defined within the current file only.
- **Shared Library:** A published collection of components and styles available across multiple files and teams for consistency.

15. What is the purpose of Figma’s Prototype Mode?

Answer:
- **Prototype Mode:** Create interactive links between frames to simulate navigation, add transitions, overlays, and micro-interactions for usability testing.
- Use prototyping to validate flows before development.

16. What is a User Flow? Give one real-world example.

Answer:
- **User Flow:** A step-by-step diagram showing screens, user actions, and decision points to complete a task.
- **Example:** E-commerce purchase flow: Home → Product Listing → Product Detail → Add to Cart → Checkout → Payment → Confirmation.

17. Why are decision points important in a User Flow diagram?

Answer:
- **Decision points** show where users choose between actions (e.g., sign-in vs continue as guest). They affect branching, error handling, and alternate paths — essential for designing edge cases and UI states.

18. What is the difference between a Happy Path and an Error State in UX?

Answer:
- **Happy Path:** The ideal scenario where user completes task without issues.
- **Error State:** Occurs when something goes wrong (form validation errors, network failures); designers must provide clear recovery and messaging.

19. What is an Empathy Map and what are its four quadrants?

Answer:
- **Empathy Map:** Tool to capture what users **Say, Think, Do, and Feel**.
- **Four quadrants:** Says, Thinks, Does, Feels — plus optional Pains and Gains as separate areas.

20. Why is the “Thinks” quadrant harder to fill than the “Says” quadrant?

Answer:
- **Says** contains direct quotes from interviews; **Thinks** requires inference about internal beliefs and motives which are not always spoken — needs triangulation from behavior and cues.

21. What is the difference between an Empathy Map and a User Persona?

Answer:
- **Empathy Map:** Snapshot of users’ thoughts, feelings, actions, and needs — used during early research.
- **Persona:** A more detailed, research-backed profile (goals, motivations, demographics) used for design decisions.

22. How does an Empathy Map help write better problem statements?

Answer:
- By surfacing **unspoken needs, pains, and gains**, empathy maps provide concrete evidence to craft focused problem statements that reflect user context and emotions.

23. What happens when you detach an Instance from its Main Component in Figma?

Answer:
- Detaching an instance breaks its link to the main component; the instance becomes a standalone group/layers and no longer receives updates from the main component.

24. What are Component Variants in Figma and why are they useful?

Answer:
- **Variants:** Group related component states (e.g., default, hover, disabled) into a single component set with properties.
- **Useful because:** They simplify component management, make switching states easier, and improve consistency and handoff.

25. What is the difference between a Main Component and an Instance?

Answer:
- **Main Component (Master):** The source of truth; edits propagate to all instances.
- **Instance:** A copy that inherits properties from main but can have allowed overrides (text, swaps); changes to main update instances.

26. What is a Design System in Figma?

Answer:
- **Design System:** A collection of components, styles, tokens, and guidelines published as a shared library to maintain consistent UI across products and teams.

27. What is the difference between Boolean Property and Text Property in Component Variants?

Answer:
- **Boolean Property:** Toggle between two states (true/false) like `checked: true/false`.
- **Text Property:** Allows different textual values to represent variant states (e.g., `size: small/medium/large` displayed as text).

28. What does the Empathize phase mean in Design Thinking?

Answer:
- **Empathize:** The phase where designers deeply observe and engage with users to understand needs, contexts, and emotions using qualitative research.

29. What is the difference between empathy and sympathy in UX Design?

Answer:
- **Empathy:** Actively understanding and sharing user perspectives to design solutions.
- **Sympathy:** Feeling pity without fully understanding the user’s context; less useful for practical design decisions.

30. Name three research methods used in the Empathize phase.

Answer:
- **User interviews, contextual inquiry (observation), and diary studies.**

31. Why do users often say one thing but do another?

Answer:
- Due to **social desirability**, memory bias, or lack of awareness; observed behavior and analytics often reveal true actions — hence the need for behavioral research.

32. What does “Beginner’s Mind” mean in empathy-driven design?

Answer:
- **Beginner’s Mind:** Approaching research without assumptions or preconceptions to notice unexpected user behaviors and insights.


## LONG ANSWERS / SCENARIO-BASED QUESTIONS

1. You are designing a mental health support app for college students.
   Explain your complete Empathize phase research plan.

Answer:
**Goal:** Understand students’ mental health needs, triggers, coping habits, and barriers to seeking help.

**Research Objectives:**
- Identify common stressors, help-seeking behaviors, preferred communication channels, privacy concerns.

**Methods & Steps:**
- **Stakeholder kickoff:** Align with counselors, campus admin on goals and ethics.
- **Secondary research:** Review studies on student mental health, campus services, and relevant apps.
- **Recruit participants:** Diverse sampling (years, streams, gender, international/local).
- **User interviews (semi-structured):** 20–30 students; questions on daily stressors, support used, mobile habits, privacy concerns. Capture quotes for empathy maps.
- **Contextual inquiry / shadowing:** Observe students during study sessions, counseling queues to see real contexts.
- **Diary study (2 weeks):** 10 participants record mood triggers, app usage, coping strategies; collects longitudinal context.
- **Surveys (quantitative):** Short scale to measure prevalence of issues, preferred features, and willingness to use app.
- **Usability probe:** Quick sketch-tests of potential features (anonymous chat, resources) to get initial reactions.

**Deliverables:**
- Personas (e.g., "Overloaded Finalist"); Empathy maps; Journey maps showing crisis and recovery paths; Problem statements (How Might We); research synthesis report with quotes and prioritized needs.

**Ethics & Safety:**
- Confidentiality, informed consent, emergency protocols for crisis disclosures, provide resource list.

**Outcome / Use:**
- Evidence-based features (anonymous chat, quick coping exercises, signposting to counseling), prioritized backlog for prototyping and testing.

**Conclusion:**
- The Empathize plan combines qualitative and quantitative methods with ethical safeguards to create real user-centered product requirements.

2. A startup is building a food delivery app for working professionals.
   (a) Describe three research activities.
   (b) Write a problem statement using “How Might We”.

Answer:
(a) **Research activities:**
- **Contextual interviews:** Meet professionals at workplaces to observe lunch habits, time constraints, and ordering patterns.
- **Competitive analysis:** Compare features, delivery times, pricing, and loyalty programs of top competitors (Zomato, Swiggy) to identify gaps and opportunities.
- **Surveys & analytics review:** Quick in-app or email surveys to measure frequency of orders, preferred cuisine, pain points; analyze order logs for peak times.

(b) **How Might We problem statement:**
- "How might we help busy working professionals quickly discover and order healthy, affordable meals that arrive reliably during short lunch windows?"

4. Differentiate between Primary Research and Secondary Research with examples.

Answer:
- **Primary Research:** Original data collection directly from users (interviews, surveys, usability tests). Example: Conducting five in-depth interviews with pilots about cockpit UI.
- **Secondary Research:** Analysis of existing sources (papers, reports, competitor apps, analytics). Example: Reviewing government transport reports to understand commuter volumes.
- **When to use:** Secondary is quick and low-cost for context; primary provides direct user insight and validation.

5. You are redesigning a hospital appointment booking system.
   Explain your complete UX research process.

Answer:
**Objective:** Reduce booking friction, no-shows, and improve patient satisfaction.

**Research phases:**
- **Stakeholder interviews:** Doctors, reception, admin to identify constraints and requirements.
- **Analytics review:** Current appointment drop-offs, busiest scheduling slots, device use statistics.
- **User interviews:** Patients across demographics (elderly, caregivers) to gather booking experiences and pain points.
- **Contextual observation:** Watch front-desk interactions, phone booking workflow, and clinic flow for real-world constraints.
- **Journey mapping:** Map touchpoints from booking to arrival and post-visit follow-up; identify pain nodes (confirmation, reminders, rescheduling).
- **Prototyping & usability testing:** Paper prototype → mid-fi clickable prototype; test with representative users for key tasks: book, reschedule, cancel.
- **Iterate & validate:** Implement changes and run A/B tests on reminder messages and UI tweaks to reduce no-shows.

**Deliverables:**
- Personas (elderly, working parent), updated user flows, low-to-high fidelity prototypes, usability findings, prioritized backlog, KPI metrics (reduced booking time, lower no-show rate).

**Conclusion:**
- Research combines stakeholder needs with user-centered testing to produce measurable UX improvements.

6. Design a two-week UX research plan for a B2B SaaS project management tool.

Answer:
**Goal:** Validate feature prioritization and onboarding flow for PM tool aimed at mid-size product teams.

**Week 1 — Discovery & Qualitative:**
- Day 1: Stakeholder kickoff and KPI alignment.
- Day 2–3: Secondary research & competitive benchmarking.
- Day 4–7: 8–10 contextual interviews with PMs and engineers; record workflows and pain points; create empathy maps and initial personas.

**Week 2 — Validation & Prototype Testing:**
- Day 8–9: Synthesize insights; define problem statements and user flows.
- Day 10: Create low-fidelity wireframes for onboarding and task management.
- Day 11–13: Remote moderated usability tests (5–8 participants) on prototypes; measure task success, time on task, SUS score.
- Day 14: Analysis, prioritized recommendations, and presentation to stakeholders.

**Deliverables:** Research report, personas, user journeys, prioritized backlog, prototype iterations.

7. Explain the difference between UI, UX, and Product Thinking using Zomato example.

Answer:
- **UI (Visual & Interaction):** Zomato’s color palette, menu layout, and prominent CTA buttons — makes app visually usable.
- **UX (Experience):** How easily a user finds restaurants, orders, tracks delivery, and handles refunds — end-to-end flow and satisfaction.
- **Product Thinking:** Strategic decisions about which features to build (e.g., subscription plans, partner integrations, location expansion) aligned with business goals and user needs.
- **Example:** UI: red order button; UX: 3-step checkout; Product Thinking: launching a subscription to increase retention and lifetime value.

8. Design a UPI payment app for small businesses.
   Explain Product Thinking, UX Design, and UI Design layers.

Answer:
**Product Thinking:**
- Define target users (kirana owners, salons), business goals (quick payments, low fees), metrics (transaction success rate, retention).
- Feature decisions: one-tap receive, invoices, QR code generator, settlement reporting.

**UX Design:**
- Map user flows: receive payment, refund, view settlements.
- Ensure error handling, offline modes, onboarding with KYC steps, and accessible layouts for low-literacy users.

**UI Design:**
- Visual language: clear typography, large tap targets, high-contrast colors.
- Microcopy: clear labels, confirmation modals, success/failure states with icons.

**Conclusion:**
- Product thinking defines the what and why; UX defines the flow and interactions; UI makes it usable and attractive.

9. Analyze why Juicero failed using Design Thinking principles.

Answer:
**Background:** Juicero sold expensive, proprietary cold-press juicers and juice packs but failed commercially.

**Design Thinking analysis:**
- **Empathize failure:** Misread user need — customers wanted convenience and health but not a $700 machine tied to consumables.
- **Define failure:** Problem framed as better-pressing tech rather than cheaper convenience or broader value (cost, portability).
- **Ideate & Prototype failure:** Focused on high-tech hardware not validated with mass users; insufficient exploration of cheaper alternatives.
- **Test failure:** Market feedback showed packs could be squeezed by hand — exposed weak value proposition.
- **Product lesson:** Lack of product-market fit, poor pricing strategy, and over-reliance on hardware made solution fragile.

**Conclusion:**
- Juicero lacked empathetic validation and rapid market testing; Design Thinking would have recommended low-cost prototypes and deeper user interviews early.

10. Explain all five phases of Design Thinking using a college canteen app example.

Answer:
- **Empathize:** Interview students and staff to learn peak hours, payment pain, menu preferences, and dietary needs.
- **Define:** Problem statement: "Students waste 20–30 minutes in queues during lunch hours and often miss classes."
- **Ideate:** Ideas: pre-order slots, express pickup, dynamic menu, subscription meals.
- **Prototype:** Paper mockups of pre-order flow, pickup locker sketch, and quick payment interface.
- **Test:** Run pilot with 50 students using the prototype; measure wait time reduction and collect feedback for iteration.

**Conclusion:**
- The iterative process lowers risk and builds a service that addresses real student needs.

11. Apply the complete Design Thinking framework for an e-commerce ethnic clothing platform with high cart abandonment.

Answer:
**Empathize:**
- Conduct interviews with shoppers to understand friction: sizing uncertainty, high shipping, return anxiety, unclear product images.

**Define:**
- Problem statement: "Users abandon carts due to uncertainty about fit, personalization, and lack of trust during checkout."

**Ideate:**
- Solutions: virtual try-on, size guides with measurements, clear return policy, trust badges, guest checkout, faster shipping options.

**Prototype:**
- Build A/B prototypes: (A) enhanced size guide + AR try-on, (B) simplified checkout + free returns banner.

**Test:**
- Run usability tests and A/B experiments to measure conversion uplift, time-to-purchase, and return rates.

**Outcome & Iteration:**
- Implement highest-impact changes (size guide and simpler checkout), monitor metrics, iterate with more personalization features.

12. Explain Nielsen’s 10 Usability Heuristics with practical examples.

Answer:
- (Provide short explanation and practical example for each heuristic.)
- 1. **Visibility of system status:** Show loading spinner and progress for uploads.
- 2. **Match between system and real world:** Use calendar icons and date formats familiar to users.
- 3. **User control and freedom:** Add undo/redo and easy exit from flows (cancel order).
- 4. **Consistency and standards:** Same icon for settings across app and website.
- 5. **Error prevention:** Validate forms in-line and disable submit until required fields are filled.
- 6. **Recognition rather than recall:** Show recently used items and auto-complete.
- 7. **Flexibility and efficiency of use:** Keyboard shortcuts for power users and quick actions.
- 8. **Aesthetic and minimalist design:** Remove redundant information from checkout screen.
- 9. **Help users recognize, diagnose, recover from errors:** Provide clear error messages with corrective steps ("Invalid card — check expiration date").
- 10. **Help and documentation:** Provide searchable FAQ and contextual tips.

13. Explain different UX Research methods with advantages and disadvantages.

Answer:
- **Interviews (qualitative):** Deep insights; time-consuming and limited sample.
- **Surveys (quantitative):** Scalable and measurable; may suffer from self-report bias.
- **Observation / contextual inquiry:** Real behavior insights; intrusive and resource-heavy.
- **Diary studies:** Longitudinal context; participant fatigue risk.
- **Usability testing:** Direct task-based metrics; small sample can find most usability issues but needs prototyping effort.

14. Describe User Journey Maps and User Flows with examples and diagrams.

Answer:
- **User Journey Map:** Timeline-style diagram showing stages (discover → consider → buy → use → support), touchpoints, emotions, and opportunities.
- **User Flow:** Flowchart of screens and decisions for a task.

Example diagrams (text):
- Journey: Onboarding -> Search -> Add to cart -> Checkout -> Delivery -> Support (with emotions tagged under each).
- Flow: Home → Search → Product → Add to Cart → Checkout → Payment → Success.

15. Explain Card Sorting, Mental Models, and Information Architecture in detail.

Answer:
- **Card Sorting:** Users group topics into categories to reveal natural information structures; helps design navigation and IA. Types: open card sort (users name categories), closed (predefined categories).
- **Mental Models:** Users’ expectations about how tasks should work; align IA and labels to users’ models to reduce cognitive load.
- **Information Architecture (IA):** Organizing content, labeling, and navigation to help users find information easily; use taxonomies, sitemaps, and wireframes.

16. Explain Usability Testing methods and important usability metrics.

Answer:
- **Methods:**
  - **Moderated lab testing:** Facilitator observes user completing tasks.
  - **Remote moderated:** Same as above via video call.
  - **Unmoderated remote:** Users complete tasks independently using testing platforms.
  - **Guerrilla testing:** Quick tests in public spaces with minimal setup.
- **Metrics:** Task success rate, time on task, error rate, System Usability Scale (SUS), Net Promoter Score (NPS), and user satisfaction ratings.

17. Explain different fidelity levels in Wireframing and Prototyping.

Answer:
- **Low-fidelity (Lo-fi):** Paper sketches or wireframes; focus on layout and flow; fast and cheap.
- **Mid-fidelity:** Digital wireframes with basic interactions; useful for early usability tests.
- **High-fidelity:** Pixel-perfect mockups and interactive prototypes that resemble final product for validation and stakeholder buy-in.

18. Explain Frames, Groups, Sections, and Constraints in Figma.

Answer:
- **Frames:** Artboards and containers for screens; support responsive constraints and Auto Layout.
- **Groups:** Simple layer grouping for organization.
- **Sections:** Visual groupings across canvas to organize related screens.
- **Constraints:** Define how child layers scale or align when parent resizes (left/right/center/stretch) — essential for responsive design.

19. Explain Auto Layout in Figma with properties and practical examples.

Answer:
- **Auto Layout:** Lets components auto-size and distribute children along an axis.
- **Key properties:** Direction (horizontal/vertical), spacing between items, padding, alignment, and resizing rules (fixed/ fill container).
- **Example:** Build a responsive button with icon + label that maintains padding and reflows when label text changes.

20. Explain Components, Variants, and Design Systems in Figma.

Answer:
- **Components:** Reusable UI elements (buttons, inputs) — edit main to update instances.
- **Variants:** Group related component states (size, color, state) into a single organized set with properties.
- **Design System:** Centralized library of styles, components, tokens, and documentation enabling consistency and scale.

21. Explain Figma Prototype Mode, interactions, animations, and overlays.

Answer:
- **Prototype Mode:** Link frames with interactions (on click, while hovering) to simulate user journeys.
- **Animations:** Transitions (instant, dissolve, smart animate) to convey motion; **Smart Animate** interpolates properties between similar layers.
- **Overlays:** Floating panels (modals, tooltips) that appear above content; can be positioned relative to click.

22. Explain Developer Handoff process in Figma.

Answer:
- **Handoff steps:** Finalize components and styles, publish library, prepare specs, document behaviors and edge cases, and share file link.
- **Figma features:** Inspect panel shows CSS, iOS/Android code snippets, assets export settings, and measurements for developers.
- **Best practices:** Use tokens, name layers semantically, maintain versioning, and include acceptance criteria.
